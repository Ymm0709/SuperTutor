# 聊天记录 - SuperTutor 开发过程

本文档记录了开发 SuperTutor 应用过程中的所有用户指令和问题。

---

## 1. 添加新题目到题库

**用户指令：**
- 添加新题目到题库
- 分配难度等级
- 提供中英文翻译

---

## 2. 优化题目选择逻辑

**用户指令：**
- 优化题目选择逻辑，确保题目从易到难呈现

---

## 3. 检查题目和答案对应关系

**用户问题：**
- "为什么我选择的是正确的却说我是错误的？检查一下所有题目和答案的对应有没有问题"

**后续发现：**
- Q18 的答案在多语言模式下不匹配
- Q20 的答案在多语言模式下不匹配

---

## 4. 添加"答案"按钮功能

**用户指令：**
- "每个用户在第一次打错后，hint左边可以在弹出来一个功能，answer 答案，每个用户重置前只能使用3次"

**功能要求：**
- 第一次答错后显示"答案"按钮
- 每个用户重置前只能使用3次
- 点击后显示正确答案

---

## 5. 防止题目重复

**用户指令：**
- "已经做打过的题要被拿出题库 直到重制之后再加进来 不能重复给用户做一道题"

**功能要求：**
- 已做过的题目要从题库中移除
- 直到重置之后再加进来
- 不能重复给用户做一道题

---

## 6. 语言切换不刷新题目

**用户指令：**
- "切换语言不更改题目 只是单纯切换语言而不是刷新"

**后续反馈：**
- "还是会改变题目啊 相当于刷新了整个页面啊 我不希望这样"

**功能要求：**
- 切换语言时只改变显示语言
- 不重新生成题目
- 不重置状态（已提交的答案、已选择的选项等）

---

## 7. 上传代码到 GitHub

**用户指令：**
- "https://github.com/Ymm0709/SuperTutor.git 上传到这里"

---

## 8. GitHub Pages 部署

**用户指令：**
- "帮我通过github部署"

**遇到的问题：**
- "ons 2 errors build HttpError: Not Found build Get Pages site failed. Please verify that the repository has Pages enabled and configured to build using GitHub Actions"

**后续确认：**
- 用户已在 GitHub Pages 设置中进行了配置

---

## 9. 关于 GitHub Pages 路径前缀

**用户问题：**
- "是不是通过github部署 就一定会有https://ymm0709.github.io/SuperTutor/ 前缀"

---

## 10. 创建聊天记录文档

**用户指令：**
- "你可以帮我把我对你的所有指令全部整合起来变成一个md文件吗，就比如1.我问了什么问题；2.我问了什么问题 不需要你是怎么回答的 只需要 chatting record"

---

## 总结

整个开发过程涵盖了：
1. 题目库管理和多语言支持
2. 题目选择逻辑优化
3. 答案验证和错误修复
4. 交互功能增强（答案按钮、防重复）
5. 语言切换优化
6. 代码版本控制和部署
